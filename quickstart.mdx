Get up and running with effect-orpc by building a complete working example with services, tagged errors, and Effect procedures.

<Steps>
  <Step title="Install effect-orpc">
    Install effect-orpc and its peer dependencies using your preferred package manager.

    <CodeGroup>
    ```bash npm
    npm install effect-orpc effect @orpc/server
    ```
    ```bash pnpm
    pnpm add effect-orpc effect @orpc/server
    ```
    ```bash yarn
    yarn add effect-orpc effect @orpc/server
    ```
    ```bash bun
    bun add effect-orpc effect @orpc/server
    ```
    </CodeGroup>

    <Note>
      You'll also need a schema validation library like `zod` for input/output validation.
    </Note>
  </Step>

  <Step title="Set up your runtime and services">
    Create an Effect service to manage your application logic. In this example, we'll create a simple user repository service.

    ```typescript
    import { Effect, ManagedRuntime } from "effect";
    import { os } from "@orpc/server";
    import { makeEffectORPC, ORPCTaggedError } from "effect-orpc";

    interface User {
      id: number;
      name: string;
    }

    let users: User[] = [
      { id: 1, name: "John Doe" },
      { id: 2, name: "Jane Doe" },
      { id: 3, name: "James Dane" },
    ];

    // Define your service
    class UsersRepo extends Effect.Service<UsersRepo>()("UsersRepo", {
      accessors: true,
      sync: () => ({
        get: (id: number) => users.find((u) => u.id === id),
      }),
    }) {}

    // Create runtime with your services
    const runtime = ManagedRuntime.make(UsersRepo.Default);
    ```

    The `ManagedRuntime` provides type-safe service injection to your procedures. Any service included in the runtime can be accessed in your Effect procedures.
  </Step>

  <Step title="Create tagged errors">
    Define custom error classes using `ORPCTaggedError` for type-safe error handling.

    ```typescript
    // Create a yieldable oRPC error class
    class UserNotFoundError extends ORPCTaggedError("UserNotFoundError", {
      status: 404,
    }) {}
    ```

    Tagged errors can be yielded directly in Effect generators and automatically integrate with oRPC's error handling system.
  </Step>

  <Step title="Create the Effect-aware builder">
    Use `makeEffectORPC` to create a procedure builder that integrates your runtime and errors.

    ```typescript
    // Create Effect-aware oRPC builder
    const effectOs = makeEffectORPC(runtime).errors({
      UserNotFoundError,
    });
    ```

    You can also base your Effect builder on an existing oRPC builder to inherit middleware, context, and other configurations:

    ```typescript
    // Optional: authenticated base builder
    const authedOs = os
      .errors({ UNAUTHORIZED: { status: 401 } })
      .$context<{ userId?: number }>()
      .use(({ context, errors, next }) => {
        if (context.userId === undefined) throw errors.UNAUTHORIZED();
        return next({ context: { ...context, userId: context.userId } });
      });

    const effectOs = makeEffectORPC(runtime, authedOs).errors({
      UserNotFoundError,
    });
    ```
  </Step>

  <Step title="Create your first Effect procedure">
    Write procedures using Effect generators with `yield*` syntax to access services and handle errors.

    ```typescript
    const getUser = effectOs.effect(function* ({ context: { userId } }) {
      // Access the service using yield*
      const user = yield* UsersRepo.get(userId);
      
      // Yield errors for type-safe error handling
      if (!user) {
        return yield* new UserNotFoundError();
      }
      
      return user;
    });
    ```

    Effect procedures work just like standard oRPC procedures but with full Effect integration, including service injection and tagged error support.
  </Step>

  <Step title="Build your router">
    Create your router by combining Effect procedures with standard oRPC procedures.

    ```typescript
    export const router = {
      // Standard oRPC procedure
      health: os.handler(() => "ok"),
      
      // Effect procedures with service injection
      users: {
        me: getUser,
      },
    };

    export type Router = typeof router;
    ```

    You can mix Effect procedures and standard oRPC procedures in the same router for maximum flexibility.
  </Step>
</Steps>

## Complete example

Here's the full working example combining all the steps above:

```typescript
import { os } from "@orpc/server";
import { Effect, ManagedRuntime } from "effect";
import { makeEffectORPC, ORPCTaggedError } from "effect-orpc";

interface User {
  id: number;
  name: string;
}

let users: User[] = [
  { id: 1, name: "John Doe" },
  { id: 2, name: "Jane Doe" },
  { id: 3, name: "James Dane" },
];

// Authenticated os with initial context & errors set
const authedOs = os
  .errors({ UNAUTHORIZED: { status: 401 } })
  .$context<{ userId?: number }>()
  .use(({ context, errors, next }) => {
    if (context.userId === undefined) throw errors.UNAUTHORIZED();
    return next({ context: { ...context, userId: context.userId } });
  });

// Define your services
class UsersRepo extends Effect.Service<UsersRepo>()("UsersRepo", {
  accessors: true,
  sync: () => ({
    get: (id: number) => users.find((u) => u.id === id),
  }),
}) {}

// Special yieldable oRPC error class
class UserNotFoundError extends ORPCTaggedError("UserNotFoundError", {
  status: 404,
}) {}

// Create runtime with your services
const runtime = ManagedRuntime.make(UsersRepo.Default);

// Create Effect-aware oRPC builder
const effectOs = makeEffectORPC(runtime, authedOs).errors({
  UserNotFoundError,
});

// Create the router with mixed procedures
export const router = {
  health: os.handler(() => "ok"),
  users: {
    me: effectOs.effect(function* ({ context: { userId } }) {
      const user = yield* UsersRepo.get(userId);
      if (!user) {
        return yield* new UserNotFoundError();
      }
      return user;
    }),
  },
};

export type Router = typeof router;
```
